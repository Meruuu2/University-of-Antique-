User Request: Configure Laravel to use MySQL in XAMPP instead of SQLite

================================================================================
ISSUE ENCOUNTERED
================================================================================

Error Message:
  "Database file at path [C:\Users\user\Desktop\project_app\database\database.sqlite] 
   does not exist. Ensure this is an absolute path to the database."

Root Cause:
  - Laravel was configured to use SQLite by default
  - .env file had DB_CONNECTION=sqlite
  - SQLite database file was never created
  - User needs to use MySQL from XAMPP instead

================================================================================
SOLUTION IMPLEMENTED
================================================================================

File Modified: .env

BEFORE:
  DB_CONNECTION=sqlite
  # DB_HOST=127.0.0.1
  # DB_PORT=3306
  # DB_DATABASE=database.sqlite
  # DB_USERNAME=root
  # DB_PASSWORD=

AFTER:
  DB_CONNECTION=mysql
  DB_HOST=127.0.0.1
  DB_PORT=3306
  DB_DATABASE=laravel_app
  DB_USERNAME=root
  DB_PASSWORD=

NEXT STEPS
================================================================================

1. ENSURE XAMPP MySQL IS RUNNING:
   - Open XAMPP Control Panel
   - Click "Start" next to "MySQL"
   - Verify "MySQL" shows as "Running"

2. CREATE DATABASE IN MySQL:
   - Option A: Using phpMyAdmin
     * Open http://localhost/phpmyadmin
     * Click "New"
     * Enter database name: "laravel_app"
     * Click "Create"
   
   - Option B: Using MySQL Command Line
     * Open terminal/command prompt
     * Run: mysql -u root -p
     * Type: CREATE DATABASE laravel_app;
     * Type: EXIT;

3. RUN LARAVEL MIGRATIONS:
   - Run: php artisan migrate
   - This will create all necessary database tables

4. VERIFY CONNECTION:
   - Refresh http://localhost:8000 in browser
   - If you see the Laravel welcome page without errors, connection is successful

================================================================================
IMPORTANT NOTES
================================================================================

- Password: If XAMPP MySQL has NO password (default), leave DB_PASSWORD empty
- If password exists, add it to .env: DB_PASSWORD=your_password
- Database Name: Can be changed to any name, just create it in MySQL first
- Connection Test: Run "php artisan tinker" then type "DB::connection()->getPDO();"
  If no error appears, connection is working

================================================================================
TROUBLESHOOTING
================================================================================

If you get "SQLSTATE[HY000] [1045]" error:
  - Check if MySQL is running in XAMPP
  - Verify username/password are correct
  - Check database exists: CREATE DATABASE laravel_app;

If you get "Can't connect to MySQL server":
  - Ensure XAMPP MySQL service is running
  - Check port 3306 is open
  - Verify no firewall is blocking MySQL

If migrations fail:
  - Delete database: DROP DATABASE laravel_app;
  - Create new one: CREATE DATABASE laravel_app;
  - Run: php artisan migrate --fresh
